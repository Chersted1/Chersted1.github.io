1.IDA pro

逆向工程中常用且好用的反汇编工具

1.1

打开ida，可以看见三个选项

New：
打开资源管理器选择一个程序进行反汇编，但由于ida支持文件拖拽，所以并不常用
Go:
最常用的选项，直接进入主界面

展示你先前已反编译的文件选项

1.2

当你将一个文件托入窗口，会显示如下对话框

Load file：
文件路径，下面为文件类型选择，ida会自行判断。如果IDA不能识别的二进制代码，可选择“Binary file”方式加载。此时下方的Loading segment 和 Loading offset 就可以进行编辑。你可以手动填入加载段地址和相对偏移，此时ida不会自动分析代码，意味着用户可根据具体需求自行反汇编二进制代码
Processor Type：
可以指定在反汇编过程中使用的处理器模块。大多数情况下，ida会从可执行文件的头中读取到信息，会自行选择合适的处理器
kernel options：
配置特定反汇编分析选项，通常ida默认就是最优选项
processor options：选择适用于处理器模块的配置选项

reverse工程

发表于 2026-05-11 分类于 基础知识 本文字数： 2.5k 阅读时长 ≈ 5 分钟

1.概念

1.1什么是逆向工程
逆向工程（Reverse Engineering）是指在没有源代码的情况下，通过对编译后的程序（如 .exe、.dll、.apk、.so 等二进制文件）进行分析，来理解其内部逻辑、算法、数据结构、通信协议或安全机制的技术过程。

1.2逆向工程有什么用
-安全研究与漏洞挖掘，通过逆向程序找出漏洞，避免安全隐患
-兼容性开发，为没有公开接口的软硬件开发驱动或插件，实现与闭源系统的数据互通
-软件质量评估与审计，对商业软件进行安全审计，检查是否存在后门、隐私泄露或弱加密

2.基础知识

2.1语言基础
- C/C++ 目前绝大多数逆向目标是 C/C++ 编写的，理解源码 → 汇编的映射关系至关重要。其中理解指针、结构体、位运算、编译后如何对应到汇编（例如数组在内存中如何排布）十分重要
- 汇编语言，逆向工程的核心。您需要至少熟悉一种体系结构的汇编，如 x86, x86-64 (AMD64), ARM。也应熟悉常用指令：mov, push/pop, call/ret, jmp, cmp/test, lea等
- python，用于编写自动化脚本（如 IDA 插件、解密工具）十分好用。常用库：pwntools, capstone（反汇编引擎）, keystone（汇编引擎）

3.工具

3.1 IDA Pro

用途：反汇编器 + 静态分析神器，是逆向工程师的“行业标准”。
功能：
-将二进制程序（如 .exe, .dll, .so）转换为汇编代码甚至类C伪代码（需配合Hex-Rays插件）。
-支持多种架构（x86, x64, ARM, MIPS 等）和文件格式（PE, ELF, Mach-O 等）。
-提供函数识别、交叉引用（XREF）、注释、图形化控制流图等强大功能。

  • 特点：功能极强，但商业收费（学生可申请免费版）；也有免费替代品如 Ghidra。

✅ 用于：静态分析恶意软件、漏洞挖掘、协议还原、CTF 逆向题。

3.2 x64dbg

用途：一个开源、免费的 Windows 平台动态调试器，主要用于动态分析。
特点：
支持 32 位（x32dbg）和 64 位程序调试。
图形界面友好，插件丰富，社区活跃。
可设置断点、查看寄存器/内存、跟踪函数调用等。
适用场景：调试加壳程序、分析运行时行为、破解验证逻辑等。

3.3 010 Editor

用途：高级十六进制编辑器 + 模板解析器。
功能：
- 以十六进制和 ASCII 形式查看/编辑任意文件（包括磁盘、进程内存）
- 内置 Binary Templates（模板）：可自动解析 PE、ZIP、BMP、ELF 等结构，像“看结构体”一样看二进制数据
- 支持脚本（C-like 语法）自动化处理

  • 优势：比普通十六进制编辑器更智能，适合分析文件格式

3.4 WinHex

用途：老牌十六进制编辑器 + 数字取证工具。
功能：
- 编辑磁盘、分区、文件、内存。
- 支持磁盘克隆、数据恢复、关键词搜索（支持正则）。
- 常用于电子取证（如从硬盘中恢复删除文件）。

3.5 Apktool

用途：Android APK 逆向工具。
功能：
- 反编译 APK：提取 AndroidManifest.xml、资源文件（图片、布局）、smali 代码（Dalvik 字节码的汇编表示）。
- 回编译：修改后可重新打包成 APK（常用于汉化、去广告、重打包）。

4.如何分析

4.1 静态分析（Static Analysis）

定义：不运行程序，直接分析其二进制代码。
常用工具：
- IDA Pro：反汇编 + 反编译（生成伪 C 代码）。
- 010 Editor / WinHex：查看/编辑二进制结构，配合模板解析文件格式。
**关键：
- 定位关键函数（如登录验证、加密算法）
- 识别字符串、常量、API 调用

4.2 动态分析（Dynamic Analysis）

定义：在程序运行时观察其行为。
常用工具：
- **x64dbg：Windows 程序调试器。
关键：
- 跟踪输入（如用户名、密码）在内存中的流向。
- 绕过反调试、解密运行时加载的代码。
- 验证静态分析的假设。

4.3算法还原与逻辑理解

  • 结合动静态分析，还原核心算法（如加密、校验、签名）。
  • 使用 Python/C 等语言重写关键逻辑，实现自动化（如模拟登录、生成合法请求）。

5.常见算法

1. 编码类算法

  • Base64
    将3字节数据编码为4个ASCII字符，常用于传输或隐藏二进制数据。

    • 特征：使用字符集 ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
    • 变种：Base32、Base58、自定义字符表的Base64

  • Hex 编码
    将每个字节转换为两个十六进制字符。

  • URL 编码 / HTML 实体编码
    用于Web场景中的参数转义。

2. 哈希算法（不可逆）

  • MD5
    输出128位（16字节）摘要，常用于校验或简单密码存储（已不安全）。
  • SHA 系列（SHA-1, SHA-256）
    更安全的哈希算法，常用于数字签名、完整性校验。
  • SM3（国密算法）
    中国国家密码管理局发布的哈希算法，输出256位。

3. 对称加密算法

  • AES（Advanced Encryption Standard）
    最常用的现代分组加密算法，密钥长度通常为128/192/256位。
  • DES / 3DES
    较老的加密标准，安全性较低，但在旧系统中仍可见。
  • RC4
    流加密算法，曾广泛用于SSL/TLS，现因漏洞逐渐淘汰。
  • TEA / XTEA / XXTEA
    轻量级分组加密算法，代码简短，在CTF中高频出现。

4. 异或（XOR）与简单混淆

  • 单字节/多字节 XOR
    最基础的混淆手段，常用于壳（packer）或恶意软件中。
  • ROT13 / Caesar Cipher
    简单位移加密，多见于入门级题目。

5. 自定义算法与混合逻辑

  • 多重嵌套：如 Base64 → RC4 → MD5 校验
  • 动态密钥生成：密钥由用户名、时间戳、硬件ID等动态生成
  • 控制流混淆：插入无用指令、跳转打乱逻辑，增加静态分析难度

6. 反调试与反虚拟机技术（虽非算法，但常伴随出现）

  • 检测调试器（IsDebuggerPresent、CheckRemoteDebuggerPresent）
  • 时间差检测（rdtsc 指令）
  • 虚拟机特征检测（注册表、设备名、CPUID）

SQL注入详解

发表于 2026-04-22 更新于 2026-04-23 分类于 技术教程 本文字数： 4.1k 阅读时长 ≈ 7 分钟

1.概念、

什么是SQL？结构化查询语言（Structured Query Language，缩写：SQL），一种数据库查询语言。

而SQL注入（SQL Injection）是一种常见的Web安全漏洞，形成的主要原因是web应用程序在接收相关数据参数时未做好过滤，将其直接带入到数据库中查询，导致攻击者可以拼接执行构造的SQL语句。

2.漏洞原理

SQL 注入的工作原理：

  1. 输入验证不足：当Web应用程序没有正确验证用户输入时，攻击者可以在输入字段中插入SQL代码。

  2. 拼接SQL语句：应用程序后端通常将用户输入与SQL查询拼接在一起，形成完整的数据库查询语句。

  3. 执行恶意SQL：如果应用程序没有对输入进行适当的清理或转义，恶意SQL代码将被数据库服务器执行。

  4. 数据泄露或破坏：攻击者可以利用SQL注入来查询、修改或删除数据库中的数据，或者执行数据库管理系统的系统命令。

例如

当用户登录网站时，通常会输入用户名和密码。

以下是一段正常的 SQL 查询代码：
SELECT * FROM users WHERE username = 'user1' AND password = 'password1';

如果攻击者输入：

用户名： admin' --
密码： anything

SQL 查询变成：
SELECT * FROM users WHERE username = 'admin' --' AND password = 'anything';
其中 – 是 SQL 的注释符号，忽略了密码条件，直接绕过了身份验证。

总结来说，注入产生的原因是后台服务器在接收相关参数时未做好过滤直接带入到数据库中查询，导致可以拼接执行构造的SQL语句

3.漏洞类型

SQL注入漏洞类型有多种分类标准

3.1按注入点参数类型分类

  1. 数字型注入（Integer-based）
    参数为纯数字，无引号包裹，例如：?id=1
    攻击示例：?id=1 AND 1=1

  2. 字符型注入（String-based）
    参数被单引号或双引号包裹，例如：?name=’admin’
    攻击需先闭合引号：?name=’ OR ‘1’=’1

  3. 搜索型注入（Search-based）
    常见于模糊查询（如 LIKE），使用 % 或 _ 通配符
    示例：?keyword=admin%’–

3.2按回显方式/反馈信息分类

  1. 显式注入（Union-based / Error-based）
    数据库错误信息或查询结果直接返回到页面
    可利用 UNION SELECT 获取其他表数据

  2. 盲注（Blind SQL Injection）
    页面无直接回显，需通过逻辑判断推断数据：

  3. 布尔盲注（Boolean-based）：根据页面“真/假”响应差异判断

  4. 时间盲注（Time-based）：通过 SLEEP() 等函数使数据库延迟响应，判断条件真假

3.3按注入技术/攻击手法分类

  1. 联合查询注入（Union Query Injection）
    利用 UNION 合并查询结果，获取额外数据

  2. 报错注入（Error-based Injection）
    利用数据库报错函数（如 MySQL 的 extractvalue()、updatexml()）将数据带出

  3. 堆叠查询注入（Stacked Queries）
    使用分号 ; 执行多条 SQL 语句（如：SELECT ...; DROP TABLE users;）
    ⚠️ 并非所有数据库都支持（如 PHP+MySQL 支持，但多数 ORM 不允许）

  4. 二次注入（Second-order Injection）
    恶意 SQL 被存储到数据库中，后续在另一功能点被调用时触发

  5. 宽字节注入（Wide-byte Injection）
    利用编码转换（如 GBK）绕过单引号转义，常见于未正确设置字符集的系统

  6. HTTP 头部注入
    注入点位于 User-Agent、Referer、Cookie 等 HTTP 请求头字段

3.4按提交方式分类

  1. GET 注入：参数在 URL 中（如 ?id=1）

  2. POST 注入：参数在请求体中（如登录表单）

  3. Cookie 注入：通过修改 Cookie 值注入

  4. User-Agent / Referer 注入：从 HTTP 头部字段注入

4.漏洞挖掘

当你看见以下特征时，说明这里很有可能存在SQL注入漏洞：

4.1URL中包含参数(尤其时数字或字符串)

例如
`https://example.com/product?id=123` 
`https://example.com/search?keyword=admin`

4.2提交表单(登录、搜索等)

登录框、搜素框、用户资料修改、评论等

4.3返回错误信息

例如
mysql的
`ERRORR 1064(42000:) You have an error in your SQL syntax`
sql server的
`Unclosed quotation mark after the character string...`

4.4响应时间

输入`' OR SLEEP(10)--`
如果页面明显卡顿10秒后才返回，表面可能支持时间盲注

5.基础注入类型

如何判断注入类型：
其核心是——闭合构造

一条常见的查询命令SELECT username,password FROM users WHERE id = ?
这里的问号可以有多种的闭合方式, $id, '$id', "$id", ($id)

那么什么是构造闭合呢？

已知我们可控的输入点是 ？也就是 $id , 当我们的输入与开发者后台设置的语句的 ' " ( 配对
例如后台为:
SELECT username,password FROM users WHERE id = "$id"

那么我们使传入的 $id = ‘1”‘, 那么后台执行则为
SELECT username,password FROM users WHERE id = "1" "

在这里我们对 1 完成了闭合构造，但是我们闭合了前序导致后续的 " 没有双引号配对，多出来的这个双引号则会导致报错

基础注入类型分为数字型注入和字符型注入
数字型注入特征：参数无引号包裹
字符型注入特征：

下面列举一些判断方式：

通过是否报错

后台实际输入 执行语句 是否报错 以及 相关解释
"1'" SELECT username,password FROM users WHERE id = "1'" "" 中为可以包含 ' ，而 1' 是一个合法的字符串 , 在查询时会先被强制类型转换为数字，不会报错
1' SELECT username,password FROM users WHERE id = 1' 这里的 ' 就没有闭合，会报错。
'1'' SELECT username,password FROM users WHERE id = '1'' 这里的 '与前序的' 闭合了但这样就留下了后序单着的 '，会报错。
通过报错信息
输入 后台执行 后台报错 解释
1" id = "1"" near '"1""' at line 1 去掉外层 SQL 的单引号，留下"1""，除去自己的输入 1"可知类型为 双引号 的 字符型注入
1' id = '1'' near ''1''' at line 1 同理，留下'1''除去自己的输入 1'可知类型为 单引号 的 字符型注入
'1 id = ''1' near '1'' at line 1 对于后台 SQL，由于 id = ''已经合法闭合，所以后面 1'反而为多出的语句，所以报错点在 1'

6.盲注

盲注是指攻击者不能直接获取数据库中的信息，需要通过一些技巧来判断或推断出数据库中的数据。

6.1布尔盲注

以下面的语句为例
$sql = "SELECT username,password FROM users WHERE id = ".$_GET["id"];

对于上述语句，如果 id 的传参如下：
id = 1 AND 1=1`

那么语句执行为：
`SELECT username,password FROM users WHERE id = 1 AND 1=1;`

这里会要求两个条件为真，一是有 `id=1` 这个值，二是 `1=1`，这两个条件当然是满足的，特别是后     面的这个条件。

那如果我让 AND 后面的条件为 `1 = 2`：
`SELECT username,password FROM users WHERE id = 1 AND '1'='2';` 

结果返回为空，因为 AND 后面的条件不满足。

那么利用这个 AND 符号我们可以尝试下面的一些方式来获取信息，如：
使用 length(）获取长度信息
利用 `SUBSTR()` 函数，逐步截取数据库中的某个数据
利用`MID()` 函数也是用于截取字符串的函数

6.2时间盲注

利用 SQL 语句的执行时间来判断 SQL 语句的真假，从而逐步推断出数据库中的数据。

`SLEEP()` 函数是时间盲注的核心，其语法为 `SLEEP(seconds)`
当语句被执行时，程序将会暂停指定秒数

`IF()` 函数是一种条件判断函数，它用于判断指定条件是否成立，并根据判断结果返回不同的值.
常 `IF` 和 `SLEEP` 两函数会一起使用

SELECT * FROM users WHERE username='admin' AND IF(SLEEP(5),1,0)`

如果数据库中不存在用户名为 `admin` 的用户，那么该语句将会立即返回结束；否则，程序将会暂停 5     秒钟后再返回结果。

由此我们可以获得数据
